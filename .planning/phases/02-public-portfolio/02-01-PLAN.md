Create query functions in `src/lib/queries/`. Each file imports `createClient` from `@/lib/supabase/server` and the relevant type from `@/lib/types/database`. Follow the pattern from 02-RESEARCH.md:

**profile.ts:**
```typescript
export async function getProfile(): Promise<Profile | null>
```
Fetches single row from `profiles` using `.select('*').single()`. Returns null on error (not throw -- graceful degradation for empty DB).

**projects.ts:**
```typescript
export async function getProjects(): Promise<Project[]>
export async function getProjectBySlug(slug: string): Promise<(Project & { images: ProjectImage[] }) | null>
```
- `getProjects`: Fetches all projects ordered by `display_order` asc, then `created_at` desc. Returns empty array on error.
- `getProjectBySlug`: Fetches single project by slug with its project_images (joined via `.select('*, project_images(*)')`) ordered by display_order. Returns null if not found.

**experiences.ts:**
```typescript
export async function getExperiences(): Promise<Experience[]>
```
Ordered by `is_current` desc (current jobs first), then `start_date` desc. Returns empty array on error.

**education.ts:**
```typescript
export async function getEducation(): Promise<Education[]>
```
Ordered by `start_date` desc. Returns empty array on error.

**skills.ts:**
```typescript
export async function getSkills(): Promise<Skill[]>
```
Ordered by `category` asc, then `display_order` asc. Returns empty array on error.

**messages.ts:**
```typescript
export async function insertMessage(data: { name: string; email: string; message: string }): Promise<{ success: boolean; error?: string }>
```
Inserts into `messages` table. Returns `{ success: true }` or `{ success: false, error: "message" }`.

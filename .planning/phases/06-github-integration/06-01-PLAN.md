2. Create `src/lib/queries/github.ts`:
   - Define the GraphQL query constant that fetches in ONE request: `user.pinnedItems(first: 6, types: REPOSITORY)` with fields (name, nameWithOwner, description, url, stargazerCount, forkCount, primaryLanguage { name, color }), `user.contributionsCollection.contributionCalendar` (totalContributions, weeks with contributionDays containing contributionCount and date), and `user.repositories(ownerAffiliations: OWNER, isFork: false, first: 100, orderBy: {field: STARGAZERS, direction: DESC})` with totalCount and nodes { stargazerCount }
   - Export `async function getGitHubData(): Promise<GitHubData | null>`
   - Read `process.env.GITHUB_TOKEN`; if missing, return null (graceful degradation)
   - Use the GitHub username from `process.env.GITHUB_USERNAME ?? 'akm-coding'`
   - Call `fetch('https://api.github.com/graphql', { method: 'POST', headers: { Authorization: bearer ${token}, Content-Type: application/json }, body: JSON.stringify({ query, variables: { username } }), next: { revalidate: 3600 } })`
   - If response not ok, log error and return null
   - Transform the raw GraphQL response into the `GitHubData` shape:
     - pinnedRepos: direct map from pinnedItems.nodes
     - totalContributions: from contributionCalendar.totalContributions
     - totalRepos: from repositories.totalCount
     - totalStars: sum of repositories.nodes[].stargazerCount
     - contributionDays: flatten weeks -> contributionDays, calculate level (0 for count=0, else quartile into 1-4 based on max count in the dataset)
   - The `transformContributions` helper should: flatMap all weeks' contributionDays, find maxCount, then map each day to { date, count, level } where level = 0 if count is 0, else Math.min(Math.ceil((count / maxCount) * 4), 4)

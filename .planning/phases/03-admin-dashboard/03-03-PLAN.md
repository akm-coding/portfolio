       - Empty state: "No experience entries yet."

    4. Create `src/app/admin/experience/[id]/edit/page.tsx`: Fetch by id, notFound() if missing, render ExperienceForm with data

       - useActionState with createSkill / updateSkill.bind(null, skill.id)
       - Fields: name (Input, required), category (Select dropdown or Input with common categories like "Frontend", "Backend", "DevOps", "Tools", "Languages")
       - Use Select component from Shadcn for category with options, plus an "Other" option that shows a custom text input
       - Submit: "Add Skill" or "Save" button, Cancel button calls onCancel
       - useEffect for toast feedback; on success, the page revalidates (server action handles revalidatePath)
       - Compact inline layout (flex row, not stacked)

    2. Create `src/app/admin/skills/page.tsx`:
       - This needs to be a combination: server component page that fetches data, client component section for interactivity
       - Create the page as a server component that fetches getSkills() and passes to a client SkillsList component
       - The client SkillsList component (can be in the same file or separate):
         - Groups skills by category
         - For each category, shows a section heading and a table of skills
         - Each skill row has: Name, Up arrow button (disabled if first in category), Down arrow button (disabled if last in category), Edit button, Delete button
         - Up/down arrows call reorderSkill(skillId, 'up'|'down') -- per user decision: up/down arrow buttons, no drag-and-drop
         - Edit button toggles inline SkillForm for that row (replaces the row content)
         - Delete button wrapped in DeleteDialog
         - "Add Skill" button at the top shows the SkillForm inline at the top of the page
         - Use icons from lucide-react: ArrowUp, ArrowDown, Pencil, Trash2
       - Empty state: "No skills yet. Add your first skill."

    **Server Actions** -- all follow the established pattern from src/actions/contact.ts: `'use server'` directive, `(prevState, formData) => Promise<{error?: string} | null>` signature, `revalidatePath` after mutations, `redirect` on success.

    1. Create `src/actions/profile.ts`:
       - `updateProfile(prevState, formData)`: Updates profile fields (full_name, title, tagline, bio, avatar_url, resume_url, email, phone, location, github_url, linkedin_url). avatar_url and resume_url come as hidden fields (set by ImageUpload component client-side). Validate full_name and title are required. revalidatePath('/admin/profile') and revalidatePath('/').

    2. Create `src/actions/projects.ts`:
       - `createProject(prevState, formData)`: Insert new project. Extract title, slug (auto-generate from title if empty: `title.toLowerCase().replace(/[^a-z0-9]+/g, '-').replace(/^-|-$/g, '')`), short_description, full_description, tech_stack (JSON.parse from hidden field), github_url, live_url, thumbnail_url, featured. Validate title required. After insert, if formData contains project_images (comma-separated URLs), insert into project_images table. revalidatePath + redirect to /admin/projects.
       - `updateProject(id: string, prevState, formData)`: Same fields as create but .update().eq('id', id). Handle project_images: delete existing, re-insert from form. revalidatePath + redirect.
       - `deleteProject(id: string)`: Delete project by id (cascade deletes images via DB). revalidatePath + redirect.

    3. Create `src/actions/experience.ts`:
       - `createExperience(prevState, formData)`: Insert with company, role, description, start_date, end_date, is_current (checkbox). Validate company and role required. revalidatePath + redirect to /admin/experience.
       - `updateExperience(id, prevState, formData)`: Same fields, .update().eq('id', id). revalidatePath + redirect.
       - `deleteExperience(id)`: Delete by id. revalidatePath + redirect.

    4. Create `src/actions/education.ts`:
       - `createEducation(prevState, formData)`: Insert with institution, degree, field_of_study, start_date, end_date, description. Validate institution and degree required. revalidatePath + redirect to /admin/education.
       - `updateEducation(id, prevState, formData)`: Same, .update().eq('id', id). revalidatePath + redirect.
       - `deleteEducation(id)`: Delete by id. revalidatePath + redirect.

    5. Create `src/actions/skills.ts`:
       - `createSkill(prevState, formData)`: Insert with name, category. Get max display_order for that category + 1 as new display_order. Validate name and category required. revalidatePath + redirect to /admin/skills.
       - `updateSkill(id, prevState, formData)`: Update name, category. revalidatePath + redirect.
       - `deleteSkill(id)`: Delete by id. revalidatePath + redirect.
       - `reorderSkill(skillId: string, direction: 'up' | 'down')`: NOT a form action (no prevState/formData). Swap display_order with adjacent skill in same category. Follow the exact pattern from 03-RESEARCH.md "Skill Reorder Pattern". revalidatePath('/admin/skills') and revalidatePath('/').

    6. Create `src/actions/messages.ts`:
       - `toggleMessageRead(id: string, isRead: boolean)`: Update is_read field. revalidatePath('/admin/messages') and revalidatePath('/admin').
       - `deleteMessage(id: string)`: Delete by id. revalidatePath('/admin/messages') and revalidatePath('/admin').

    **All server actions must:** use `await createClient()` from '@/lib/supabase/server', call revalidatePath for both admin and public pages (/ for home), log errors with console.error, return `{ error: '...' }` on failure.

    **Extend Query Functions** -- add mutation helpers alongside existing read functions:

       - Add `getUnreadMessageCount()`: Return count of unread messages (number).

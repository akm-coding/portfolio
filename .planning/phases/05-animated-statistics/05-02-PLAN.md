1. Create `src/components/sections/statistics-section.tsx` with a **server component wrapper** and a **client component** for the animation:

   **Server part (StatisticsSection):**
   - Async server component that calls `getStatistics()`
   - If no statistics returned (empty array), return `null` (hide section entirely per user decision)
   - Renders the client component passing the statistics data

   **Client part (AnimatedStatistics / CountUp):**
   - "use client" component that receives `statistics: Statistic[]`
   - Uses a single `useInView` ref on the section container with `once: true` and `margin: "-100px"` to detect when section enters viewport
   - For each statistic card, renders a `CountUp` sub-component

   **CountUp component:**
   - Props: `to: number`, `suffix: string | null`, `isInView: boolean`
   - Uses `useMotionValue(0)` to create an animatable value
   - Uses `useTransform(motionValue, (latest) => Math.round(latest))` to get rounded integer
   - When `isInView` becomes true, calls `animate(motionValue, to, { duration: 3, ease: "easeOut" })` -- ~3 seconds, smooth ease-out, NO bouncy overshoot per user decision
   - Subscribes to rounded value changes via `rounded.on("change", callback)` and updates DOM directly via ref (`ref.current.textContent = \`\${latest}\${suffix || ''}\``)
   - CRITICAL: The suffix MUST be included in every textContent update so "+" appears during count-up, not after (per user decision and research pitfall #2)
   - Initial render: `<span ref={ref}>0{suffix || ''}</span>`

   **Card layout (per user decisions):**
   - No section heading -- numbers speak for themselves
   - Each card is a vertical stack: icon on top, number in middle, label at bottom
   - Icon rendered with `DynamicIcon` from `lucide-react/dynamic` -- size ~32-40px, use `text-primary` color
   - Number uses large bold font (text-3xl or text-4xl font-bold)
   - Label uses muted smaller text (text-sm text-muted-foreground)
   - Cards have subtle border/shadow: `border rounded-lg p-6 shadow-sm` (consistent with project card styles)
   - All cards start counting simultaneously (single useInView controls all)

   **Responsive layout (per user decision STAT-04):**
   - Use CSS grid: `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-4 gap-6`
   - Wraps naturally if more than 4 stats exist
   - Section container: `py-16 px-4 max-w-6xl mx-auto` (match existing section spacing)

   **Important anti-patterns to avoid:**
   - Do NOT use `useState` + `setInterval` for counting -- use Motion's useMotionValue (no re-renders)
   - Do NOT import from `framer-motion` -- use `motion/react`
   - Do NOT wrap in `ScrollReveal` -- cards are always visible, only numbers animate
   - Do NOT use `{rounded}` in JSX directly -- motion values don't trigger re-renders; use ref + on("change") pattern

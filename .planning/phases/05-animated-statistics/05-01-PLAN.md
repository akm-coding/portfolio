   - Table: `statistics` with columns: id (UUID PK default gen_random_uuid()), label (TEXT NOT NULL), value (INT NOT NULL), suffix (TEXT nullable), icon_name (TEXT NOT NULL DEFAULT 'hash'), display_order (INT DEFAULT 0), created_at (TIMESTAMPTZ DEFAULT now()), updated_at (TIMESTAMPTZ DEFAULT now())

2. Add `Statistic` interface to `src/lib/types/database.ts`:
   ```typescript
   export interface Statistic {
     id: string
     label: string
     value: number
     suffix: string | null
     icon_name: string
     display_order: number
     created_at: string
     updated_at: string
   }
   ```

3. Create `src/lib/queries/statistics.ts` following the skills.ts pattern:
   - `getStatistics()`: select all ordered by display_order ascending, return Statistic[]
   - `getStatisticById(id: string)`: select single by id, return Statistic | null

4. Create `src/actions/statistics.ts` following the skills.ts pattern:
   - `createStatistic(prevState, formData)`: validate label (required), value (must be number), icon_name (required). Get max display_order, insert with display_order = max + 1. Revalidate /admin/statistics and /. Redirect to /admin/statistics.
   - `updateStatistic(id, prevState, formData)`: same validation, update by id. Revalidate and redirect.
   - `deleteStatistic(id)`: delete by id. Revalidate /admin/statistics and /.
   - `reorderStatistics(orderedIds: string[])`: receive array of IDs in new order, update each with display_order = index. Use Promise.all for batch update. Revalidate /admin/statistics and /. This is NOT a form action -- it's called directly from the drag-and-drop handler.

Run the migration against the local Supabase instance:
```bash
npx supabase db push
```
If supabase CLI is not available or local instance not running, note it for manual execution but do not block on it.
